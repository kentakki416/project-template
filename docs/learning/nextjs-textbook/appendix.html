用語集

用語意味（この本の文脈で）
Server Component (RSC)サーバーでのみ実行される既定のコンポーネント。async で直接データ取得でき、JS はクライアントに送られない。
Client Component"use client" を付けた、ブラウザで動くコンポーネント。イベント・state・ブラウザ API を使える。
島（island）Server の海の中に置く小さな Client の島。境界は葉に押し下げるほど良い。
Server Action"use server" 関数。フォーム送信等の mutation をサーバーで実行。GET には使わない。
Route Handlerroute.ts の GET/POST…。HTTP エンドポイント。Client からの動的取得やコールバックに。
proxy.tsNext.js 16 で改名された middleware。全リクエスト前段の関所（Edge ランタイム）。
動的レンダリングリクエストごとにサーバー実行。cookies()/headers() 使用で自動的にこうなる。
ISR / revalidate静的生成しつつ N 秒ごとに再生成。共通・低頻度データ向け。
ストリーミング<Suspense> で速い部分を先に返し、遅い部分を後から流す。
サーバ状態 / クライアント状態前者は DB/API 由来（真実はサーバー）、後者は UI 固有（検索語・開閉など）。Zustand は後者用。

ファイル規約チートシート

app/<segment>/ page.tsx # このURLの本体（既定 Server） layout.tsx # 配下共通の枠（再マウントされない） template.tsx # 枠だが遷移ごと再マウント loading.tsx # 配下の読込中に自動表示（=Suspense） error.tsx # 配下の throw を捕捉（"use client" 必須） not-found.tsx # notFound() / 未定義URL route.ts # HTTPエンドポイント（page と同居不可） actions.ts # そのページの Server Action（"use server"） [id]/ # 動的セグメント → params.id（await 必須） (group)/ # URLに出ないグループ（共通レイアウト用） app/global-error.tsx # ルートlayout自体の砦（ルートのみ） proxy.ts # middleware（全リクエストの関所）

Server / Client 判断フロー

  1. まずすべて Server Componentで書き始める（"use client" を書かない）。
  2. 次のどれかが要るか？ → onClick/onChange、useState/useEffect/useRef、window/localStorage、Zustand、DOM 依存ライブラリ。
  3. 要るなら、その部品だけを "use client" の小島に切り出す（ページ全体には付けない）。
  4. Client の中で Server の中身を出したい → import せず children で渡す。
  5. データは？ → 初期表示は Server Component で取得、mutation は Server Action、クライアント動的取得は Route Handler。
  6. 状態は？ → サーバ状態はサーバーに任せる。クライアント状態は useState → (共有なら) Zustand、(URL共有なら) searchParams。
