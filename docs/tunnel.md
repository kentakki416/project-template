# Expo Go 接続方式: なぜ `--tunnel` が必要だったのか

## 背景

Expo Go での開発時、通常モード（LAN接続）ではアプリが接続できず、`--tunnel` モードでのみ接続に成功した。

## Expo Go の接続方式

Expo には3つの接続方式がある。

### 1. LAN モード（デフォルト）

```
スマホ ---(Wi-Fi/LAN)--> PC (192.168.x.x:8081)
```

- スマホから PC のローカル IP に**直接接続**する
- PC 上の Metro Bundler（ポート 8081）に HTTP リクエストを送り、JS バンドルを取得する
- **前提条件**: スマホと PC が同一ネットワーク上にあり、端末間通信が許可されていること

### 2. Tunnel モード（`--tunnel`）

```
スマホ ---(インターネット)--> ngrok サーバー ---(インターネット)--> PC
```

- ngrok を使ってインターネット上にトンネルを作成する
- スマホは `https://xxxx.ngrok-free.app` のような公開 URL 経由で PC に接続する
- PC とスマホが同一ネットワーク上にある必要がない
- **前提条件**: PC がインターネットに接続できること

### 3. Local モード

```
スマホ ---(USB)--> PC (localhost:8081)
```

- USB 接続経由で localhost に接続する
- ネットワーク不要

## 今回の環境で LAN モードが失敗した原因

### Mac のファイアウォール

```bash
$ sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate
Firewall is enabled. (State = 1)
```

Mac のアプリケーションファイアウォールが有効になっており、**外部からの node プロセスへの接続（ポート 8081）がブロック**されていた。

通常であれば以下のいずれかで解決できる：

- ファイアウォールを一時的に無効化する
- node を許可リストに追加する

しかし、このMacは**管理対象Mac（Managed Mac）** であり、コマンドラインからファイアウォール設定を変更できなかった：

```bash
$ sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off
Firewall settings cannot be modified from command line on managed Mac computers.
```

### なぜ Tunnel モードなら接続できたのか

LAN モードとTunnel モードでは、**通信の方向が異なる**。

#### LAN モード（ブロックされた）

```
スマホ → [ファイアウォール] → PC:8081  ← ここでブロック
```

- スマホが PC のポート 8081 に**インバウンド接続**する
- ファイアウォールはインバウンド接続をブロックする

#### Tunnel モード（成功した）

```
PC → ngrok サーバー（アウトバウンド接続、ファイアウォール通過）
スマホ → ngrok サーバー（インターネット経由）
```

- PC が ngrok に対して**アウトバウンド接続**を確立する
- ファイアウォールは通常、アウトバウンド接続はブロックしない
- スマホは ngrok の公開 URL に接続し、ngrok がそのトラフィックを PC に中継する
- PC 側から見ると、すべて「外向き」の通信なのでファイアウォールに引っかからない

## まとめ

| 項目 | LAN モード | Tunnel モード |
|---|---|---|
| 通信方向（PC視点） | インバウンド | アウトバウンド |
| ファイアウォールの影響 | ブロックされる | 影響なし |
| 同一ネットワーク必須 | はい | いいえ |
| 速度 | 高速 | やや遅い（ngrok経由） |
| インターネット接続 | 不要 | 必要 |

### 恒久的な対策

1. **システム設定 GUI からファイアウォールに node を許可する** — 管理者権限があればGUIから設定変更できる場合がある
2. **MDM 管理者に依頼** — ポート 8081 または node プロセスの許可ルールを追加してもらう
3. **`--tunnel` を常用する** — 上記が難しい場合、開発時は常に tunnel モードを使う
